🔐 Authentication Guide

Learn how to authenticate with the Audio Library API

🎯 Quick Start

Step 1: Sign In

First, you need to get an access token by signing in:

POST /api/v1/auth/signin Content-Type: application/json { "email": "your-email@example.com", "password": "your-password" }

Step 2: Copy Access Token

From the response, copy the accessToken:

{ "message": "Signin successful", "user": { ... }, "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...", "sessionId": "uuid-here" }

Step 3: Authorize in Swagger

In the API documentation:

  1. Click the "Authorize" button (🔒) at the top right
  2. Paste your access token (without "Bearer " prefix)
  3. Click "Authorize"
  4. Click "Close"

Step 4: Make Authenticated Requests

Now you can test any protected endpoint! The token will be automatically included.

📋 Manual Testing (Postman/Curl)

If you're using Postman or curl, include the token in the Authorization header:

Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

Example with curl:

curl -H "Authorization: Bearer YOUR_TOKEN_HERE" \ http://localhost:3000/api/v1/sessions

⚠️ Common Issues

401 Unauthorized?
  • Make sure your token is not expired (1 hour limit)
  • Don't include "Bearer " when pasting in Swagger
  • Use the refresh token endpoint if your access token expired
💡 Pro Tips:
  • Access tokens expire after 1 hour
  • Use the refresh token endpoint to get new access tokens
  • Swagger UI will remember your token for the session
  • You can manage multiple sessions in the /sessions endpoint
